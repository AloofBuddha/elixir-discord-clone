## What this project is

A Discord clone built with Elixir + Phoenix LiveView, being developed as a learning project. The goal is working real-time messaging, benchmarking against Node.js, and production deployment on Fly.io. See `LEARNING.md` for the running tutorial document.

## Commands

```bash
mix phx.server          # start dev server on localhost:4000
iex -S mix phx.server   # start server + interactive REPL
mix test                 # run all tests
mix test path/to/test.exs:42  # run single test at line
mix test --failed        # re-run only failed tests
mix format               # format all .ex/.exs/.heex files
mix ecto.migrate         # run pending migrations
mix ecto.rollback        # roll back last migration
mix ecto.reset           # drop, recreate, migrate, seed
mix precommit            # format + compile + test (run before pushing)
```

## Architecture

The codebase follows Phoenix's strict domain/web split:

- `lib/discord/` — domain layer. No knowledge of HTTP. Business logic, Ecto schemas, contexts.
- `lib/discord_web/` — web layer. Controllers, LiveViews, templates, router, plugs.

**Contexts** (`lib/discord/accounts.ex`, future: `chat.ex`, `servers.ex`) are the only public API between web and domain. Controllers and LiveViews call context functions; they never touch schemas or Repo directly.

**Auth** was generated by `mix phx.gen.auth`. The key assign is `@current_scope` (not `@current_user`). Access the user as `@current_scope.user`. See `AGENTS.md` for full auth routing rules — they are strict and must be followed exactly.

## Key files to know

- `lib/discord/application.ex` — supervision tree, defines boot order
- `lib/discord_web/router.ex` — all routes; auth pipeline guards live here
- `lib/discord_web/user_auth.ex` — plugs for session/auth
- `priv/repo/migrations/` — all DB migrations; never edit after running

## Dev infrastructure

- Database: PostgreSQL (local), credentials in `config/dev.exs`
- Email: Swoosh local adapter — captured at `http://localhost:4000/dev/mailbox`
- BEAM dashboard: `http://localhost:4000/dev/dashboard`
- Formatting: `mix format` is the only formatter; no config needed

## Deployment target

Fly.io. flyctl is installed at `~/.fly/bin/flyctl`. Not yet deployed.
